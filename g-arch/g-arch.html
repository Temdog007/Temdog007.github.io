Game ARCHitecture (G-ARCH)

This is a tool to help architecture your games using an Entity Component System. This tool not to be used to develop the game. Rather, it is designed to help generate code and model a game's architecture.

How it works:

There your 5 types of objects that need to be defined.

  • Component - A set of data
  • Family - A group of entities where the constraint is based on the components
  • Entity - Belongs to one or more families. Contains one or more components.
  • System - A function to act upon all entities in a family.
