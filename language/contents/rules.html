Rules

  • Functions
    1. Functions must be first-class types
    2. All functions must be pure
    3. Functions may not take a struct as a parameter
    4. All functions can accept a sequence of its parameters or a sequence of parameters that make up an interface
  • Structs
    1. All structs must implement an interface. Any struct that doesn't have an interface implemented will have one created and implemented automatically by the transpiler.
    2. No abstract classes
    3. There are no access modifiers
    4. There are no struct methods. Only extensions for interfaces.
    5. There are no constructors.
    6. All associated struct values must be fields unless decorated with an attribute. Then, it is a property
    7. Fields in structs can be decorated with attributes to define how the property setter/getter should be defined
    8. Structs can be marked as poolable to prevent the need for allocating new objects
  • Interfaces
    1. Interfaces can only have readonly values defined. No methods
    2. Interfaces can have "method-like" calls similar to how extensions work in C#
    3. Interfaces automatically extend other interfaces that have the same fields
  • Memory
    1. No garbage collection
    2. Objects are freed as soon as they go out of scope unless marked as poolable
    3. Poolable objects are automatically pooled when they out of scope.
  • Misc
    1. Allow for automatic dependency injection
