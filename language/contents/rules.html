Rules

  • Functions
    • Functions must be first-class types
    • All functions must be pure
    • Functions may not take a class as a parameter
  • Structs
    • All structs must implement an interface. Any struct that doesn't have an interface implemented will have one created and implemented automatically by the transpiler.
    • There are no access modifiers for struct fields
    • There are no struct methods. Only extensions for interfaces.
    • There are no constructors.
    • All associated struct values must be fields unless decorated with an attribute. Then, it is a property
    • Fields in structs can be decorated with attributes to define how the property setter/getter should be defined
  • Interfaces
    • Interfaces can only have readonly values defined. No methods
    • Interfaces can have "method-like" calls similar to how extensions work in C#
    • Interfaces automatically extend other interfaces that have the same fields
